Welcome to Vieb

Intro

Welcome to Vieb's official, complete and offline documentation. You can ALWAYS come back to this page using F1 from ANY mode. The documentation consists of a single page, separated into sections. These sections can be accessed using the table of contents, or by using the help command with the section as the first argument. Please make sure to read the first chapter called "Basics" to get started with Vieb, which will explain what commands are and why there is no mouse interaction. Use these steps to scroll the webpage:
  • Press Escape once to make sure you are in normal mode.
  • Now scroll up and down with k and j.

Vieb is short for "Vim Inspired Electron Browser" and can be found on github.

Contents

Basics

Interaction with Vieb is heavily focused on the keyboard, and is mostly inspired by the popular text editor Vim. If you think a key is mapped to a weird keyboard combination, it's probably because it's also weird in Vim. Clicking links, scrolling, navigating, and most other tasks have dedicated keyboard shortcuts. The most important ones will be explained in this chapter. Single key actions will be listed like this f, modifiers such as shift will be displayed as: Shift with f.

modes

Vieb uses different modes to separate functionality. By default, you should be in normal mode, but you can go back to normal mode at any time by pressing Escape. The mode is always prominently displayed in the navbar, between the Vieb logo and the url input. Modes can be switched to by pressing a certain key from normal mode. Aside from normal mode, the following modes are available:
  • Nav - Upon pressing e you can enter a url to navigate to, and press Enter to navigate or as always Escape to cancel. The navigation bar will change color when text is entered. An orange border indicates the user input will be used to search with the search engine. A cyan border indicates that the entered text is detected as a website, which will result in a direct navigation. Suggestions from your browsing history will appear when typing. Webpages you visit often or those that match more closely with your search will appear at the top of the list. A green border is shown when a suggestion is selected, which also results in direct navigation. You can switch between the suggestions with Tab and Shift with Tab, after which you can select a suggestion by pressing Enter.
  • Insert - This mode is used to interact with the website normally. It brings back mouse interaction and passes all keyboard inputs to the website. If the cursor is pointing at a url, the url will be shown at the bottom of the screen. However, insert mode should not be overused in Vieb, because a lot of the advantages Vieb has to offer are unavailable in this mode. If you really need or want to enter insert mode, press i. The follow(ing) mode describes a much faster way to follow links or buttons.
  • Follow - Upon pressing f all links and buttons on the page will be assigned a key. When pressing the key next to a link, Vieb will click on the link. If there are a lot of links on the page, sometimes two keys are needed. You don't need to press these together, but simply after each other. To open links in a new tab, use Shift with f instead of f. Some buttons and other type of links can't be opened in a new tab. The different colors indicate the different type of links:
    • Regular link to the follow mode chapter will be outlined in blue.
    • Checkboxes and buttons will be outlined in red.
    • Text input fields will enter insert mode and will be outlined in green.
    • Elements with javascript event handlers will be outlined in orange.
    Press f to see the difference for the examples above. Remember that you can always go back to normal mode with Escape.
  • Command - Some functionality is accessed by executing commands. These can be entered after pressing Shift with ; (also known as :). A very important command is the quit command, as it's the only way to properly quit Vieb. For more detailed instructions on how to use this command, please refer to the dedicated paragraph here.
  • Search - This mode can be entered with /. After entering a string, press Enter to search for it. You can now press n or Shift with n to find next or previous matches. Searching is by default case-sensitive. When focussed on an element after searching, you can press Enter again to click on it.
  • Cursor - In this mode a blinking cursor will appear which can be moved around using Vim keybindings. If can also be used to hover over links or to scroll inside smaller scrollboxes. It's also possible to save images or preview url locations from this mode. It can be entered by pressing c. The complete list of movement options can be found here.
  • Visual - This mode can only be entered from cursor mode using the v key. It can be used to select and copy text using the keyboad. When switching from cursor mode to visual mode, the exising location will be saved as the starting point for the selection. The cursor can then be moved freely similar to cursor mode, with text being selected between the two locations. Similar to all other modes, you can exit this mode using Escape, or by pressing y to copy the selected text.
Now that all the different modes have been explained, a separate paragraph will explain how to use the basics of normal mode.

normal mode

Most functionality is accessed from normal mode using the keyboard. It offers a large list of shortcuts to interact with the website. Aside from the keys to switch modes mentioned in the previous paragraph, these are the most important when in normal mode:
  • Moving around with a single press of h j k l.
  • Zoom in and out using Ctrl with + or Ctrl with -.
  • Reset to the normal zoom level by pressing Ctrl with 0.
  • Go back and forward in history using Shift with h and Shift with l.
  • Open a new tab with t, which will enter nav mode.
  • Switch between tabs with w and b or using Shift with j and Shift with k.
  • Reload the page with r and without cache using Shift with r.
  • Close a tab with the d key.
  • Reopen a previously closed tab with u.
When you are done mastering these, please take a look at the complete list of them here.

  • You can ALWAYS go back to normal mode with Escape.

Settings

Vieb has a lot of settings, including custom keybindings. The basics chapter assumes the keybindings mentioned are unchanged. This is because most users won't need to change them, and because it would make the basics chapter a lot more complex than it needs to be. There are two ways to change the settings:
  • Change the defaults using a viebrc.json file.
  • Change a setting for just the current session, with :set.
The settings file is loaded on startup, and can be reloaded at anytime with the `:r` or `:reload` command. All settings in this file are optional and will override the default setting. Alternatively, settings can be changed for the current session with the `:set` command. For example, to use system notifications for as long as Vieb is running, open command mode with `:` and enter this command: 
set notification.system true
Vieb's set command syntax is different as compared to regular Vim, and always has the form of "set setting value". The settings file is case sensitive, just like the set command arguments. The keybindings and startup pages can only be changed with the settings file and not with the set command. Running the `:reload` command will reset any prior `:set` commands, because all settings will be reset and reapplied from the settings file.

viebrc.json

In this file you can change settings and the keybindings of Vieb. By default, this file does not exist and the original value for all settings will be used. Depending on your operating system, the location of this file will be different:
  • Windows - %APPDATA%\Vieb\viebrc.json
  • Mac - ~/Library/Application Support/Vieb/viebrc.json
  • Linux - ~/.config/Vieb/viebrc.json
The same folder is used for all other information that Vieb stores, which means there should be electron/chromium related files there. You can create the viebrc.json file there if you haven't already. Your browsing and download history will also be saved in this folder by default. See the history chapter for more details and instructions how to configure the related settings.

syntax

The syntax of the settings file is json, as the name would suggest. This is very different from Vim for the following reasons:
  • Writing a vimscript parser would be a tedious task
  • Most settings for Vim aren't present in Vieb and vice versa
  • Support for it would never be as good as Vim
Basically, there are little advantages here staying true to Vim, while at the same time adding a lot of extra work. JSON can be parsed natively by JavaScript and is easy to understand, which is why all configuration of Vieb is done with it.

keybindings

The viebrc is is the only place to change the keybindings. Changing them can be done by overwriting existing ones, or by adding completely new bindings. To remap a certain action, simply overwrite the old one with "" and create a new one with the same functionality. The identifiers for the keys are Vieb specific, and are a combination of the active modifiers and the JavaScript key.code. There is a third-party website where you can check the code for the key being pressed, which can be found here. Make sure to enter insert mode and look at the "event.code" field. Once you know the keycode, you can combine it with the modifiers like so:
  • C-KeyQ means to press Ctrl with q.
  • CS-KeyT means to press Ctrl and Shift with t.
  • F1 means to only press F1 without any other key.
  • CSMA-F2 means to hold Ctrl, Shift, Meta and Alt while pressing F2.
Please note that the order of the modifiers is important, and that they are always in uppercase. This means that sc-KeyT and SC-KeyT won't work, but CS-KeyT will.

  • Toggle between nav and normal mode with F6, similar to regular browsers (keybindings.nav.F6 and keybindings.normal.F6)
  • Escape can not be used to exit insert mode, but Ctrl with [ will still work (keybindings.insert.Escape)
  • Ctrl with q can be used to quit Vieb when in normal mode (keybindings.normal.C-KeyQ)
  • Https connections will now be downgraded to http if the server has no certificate (redirectToHttp)
  • The search engine will be google instead of the default duckduckgo. Please note that the search string will be pasted at the end of the search setting (search)
  • Change the search mode to be case-insensitive (caseSensitiveSearch)
  • Don't clear the cache when quiting Vieb (clearCacheOnQuit)
  • Clear all cookies when exiting Vieb (clearCookiesOnQuit)
  • Clear the localstorage when exiting Vieb (clearLocalStorageOnQuit)
  • Don't suggest/autocomplete commands when in command mode (suggestCommands)
  • Allow the entering of follow mode during page loads, even if elements might move around (allowFollowModeDuringLoad)
  • Increase the font size of all UI elements to 20px instead of 14px (fontSize)
  • Stop the digit keys from repeating actions (digitsRepeatActions)
  • Ads and trackers will still be blocked, but the lists will be automatically updated (adblocker)
  • This will stop the tabs from opening next to the current one using t, Ctrl with t will do the opposite of the current setting (addTabsNextToCurrentOne)
  • Reduce the notification duration to 3 seconds instead of 6 (notification.duration)
  • Display the notification in the bottom left instead of the bottom right (notification.position)
  • Downloads will be stored in the documents folder, except if method is set be "ask" (downloads.path)
  • Change the download method to "confirm" instead of "automatic" (downloads.method)
  • Automatically remove downloads from the download history once they are completely downloaded (downloads.removeCompleted)
  • Clear the entire download history when quiting Vieb (downloads.clearOnQuit)
  • Don't suggest previously visited websites from the browsing history when in nav mode (history.suggest)
  • Clear the browsing history when exiting Vieb (history.clearOnQuit)
  • Stop adding newly visited webpages to the browsing history (history.storeNewVisits)
  • Don't restore the previously open tabs when restarting Vieb (tabs.restore)
  • The recently closed tabs will no longer be restorable with the u key (tabs.keepRecentlyClosed)
  • Open this help page on startup, which can be expanded with other urls (tabs.startup)
  • The position of the window when not maximized will not be remembered and restored after restarting (windowstate.position)
  • The size of the window when not maximized will not be restored after restarting (windowstate.size)
  • After restarting, Vieb will no longer remember if the window was maximized (windowstate.maximized)
Please note that the download, tab and browsing history are separately configured. Therefor it's possible to keep and restore the open tabs without keeping the other browsing history. More information about the history settings can be found in the history chapter. To find the right setting for you, the set command can be used at runtime to test a specific setting. This concludes the documentation regarding the viebrc file. In the next paragraph, the set command will be explained.

set command

The set command can be used to change the settings after startup. When starting Vieb, the settings file is loaded and merged with the defaults. After that, most settings can be changed using the set command. For example, to use system notifications instead of Vieb's own notification system, go to command mode using Shift with ; (or :) and enter this command: 
set notification.system true
The set command always takes two arguments; one for the setting name and the other for the new value. The set command along with other commands has autocompletion by default, which helps with finding the right setting name. Settings can be changed as many times as you like, and can be reset and reloaded from the settings file with: 
reload
or just "r" for short. If the settings file was not changed after startup, running this command will results in the settings being reset to startup configuration. This means that it's a fast way to undo any set commands, but it can also be used to change the keybindings at runtime. To achieve this, simple edit the viebrc.json after Vieb was started, and use the reload command to apply all changes. Vieb only reads the settings file on startup and with the reload command, and never writes to the file as the set command is for runtime only. This means it's always safe to edit the viebrc file.

validation

The set command can be a useful tool to help with finding the right settings. Unlike the settings file, which will simply ignore errors and use the default, the set command informs the user of any mistakes made. For example, to disable the case sensitive search, the correct command would be: 
set caseSensitiveSearch false
If we would try to set the value to anything other than true or false, a warning would be displayed that a boolean is expected. All arguments are case sensitive, similar to the settings file. Lastly, if a command does not show a notification, it means that the command was successfully executed and applied.

Startup

By default, all the tabs from the previous session will be opened (tabs.restore), along with the configured startup tabs (tabs.startup). This behaviour can be changed in settings as listed between the brackets. By default, your tabs will also be stored when closed (tabs.keepRecentlyClosed), so you can reopen them with u. When "tabs.restore" is disabled, but "tabs.keepRecentlyClosed" is enabled, all your open tabs will be moved to the recently closed tab list when quiting Vieb. When both settings are set to false, no tab history is preserved, and no closed tabs can be reopened as a result. These settings do not influence the other history settings, just like the history settings don't change the tab and startup behaviour. You can learn more about the history settings in the history chapter.

arguments

Vieb can be started without any command-line arguments, but there are some available to use. First there is "--help" to display information about all the startup arguments. There is also a "--version" argument, to display the version and exit. Lastly, you can provide "--debug" or "--console" to open the Vieb debug console, which is the internal console of Vieb and not the one used to debug websites. For the website console, see the devtools command. Arguments not starting with "-" will be parsed and opened as a url. These urls can take any form if you include the protocol/schema, or will automatically be prefixed with "https://". You can provide as many urls as you want, as each of them will be opened in a separate tab, similar to the tabs.startup setting.

examples

To get a better understanding of these arguments, there are a couple of examples shown in this paragraph. If you start Vieb with with npm, make sure you have already done "npm install" and run the following: 
npm start -- --help
This will passs the "--help" argument to Vieb, which should now display all the available startup arguments. Note that with npm it's required to separate the npm arguments from the Vieb arguments, by adding the extra "--" between them. To add arguments to a regular Vieb executable, simply add them like so: 
vieb --help
Where "vieb" should be the name of the executable, such as "Vieb.exe" on Windows. To open two websites on startup with Vieb's internal console open, you can start vieb with the following arguments: 
        vieb --console "http://example.com" "github.com/Jelmerro/Vieb"
Note that the github url will be prefixed with "https://", because we omitted the schema/protocol from the url.

Normal mode

In the basics chapter the most useful keyboard shortcuts are mentioned, but this chapter lists all of the default normal mode actions available:
  • F1 Opens this help page, similar to the help command.
  • b and w to switch to the previous and next tab.
  • Alternatively, Shift with j and Shift with k can be used to switch tabs.
  • Go to nav mode with e.
  • Start the regular follow mode for the current tab with f.
  • Use F to open supported links in a new tab. The follow chapter explains the differences.
  • Go to the top or the bottom of the page using Shift with g or g g.
  • Jump to insert mode at the first text input by pressing g i.
  • Move around on the page with the keys h j k l.
  • Enter insert mode with i.
  • Reload the page with or without cache using r and Shift with r.
  • Open a new tab and enter nav mode in the new tab with t. By default this binding will open a new tab next to the current one. The position of the new tab can be configured with the setting (addTabsNextToCurrentOne).
  • Ctrl with t can be used to open the tabs at the alternative position without changing the setting each time. By default this will open the new tab at the end of all the open tabs. If the alternative position (at the end instead of next to the current one) is preferred, you can change it by setting addTabsNextToCurrentOne to false.
  • Open a new tab with the current website location prefilled in nav mode using Shift with t.
  • Close the current tab with d.
  • Reopen previously closed tabs with u. The position of the restored tab can be configured with the setting (addTabsNextToCurrentOne).
  • Switch to search mode with /.
  • Find the next or previous match when searching using n and Shift with n.
  • When a search match is selected, press Enter to click on the selected search.
  • Go forward or backwards in the tab browsing history using Shift with l and Shift with h.
  • Alternatively, you can use Ctrl with i and Ctrl with o to navigate the history.
  • Scroll half a page up or down using Ctrl with u or Ctrl with d.
  • Or an entire page by pressing Ctrl with b and Ctrl with f.
  • To scroll an entire page horizontally, use ^ and $ (shift 4 and 6).
  • Go to command mode using Shift with ; (also known as :).
  • Zoom in or out by pressing Ctrl with + and Ctrl with -.
  • Reset the zoom to 100% again using Ctrl with 0.
  • Enter cursor mode using c, s or v.
Aside from these, any number of digits can be entered before executing an action, which will result in the action being executed for the number of times entered. For example, pressing 2 and then j will result in scrolling down twice. This can be disabled by overwriting the action for any of the digits. To disable the repeat action behaviour entirely, set "digitsRepeatActions" to false in the settings. This repeat action behaviour is also present in cursor and visual mode.

Follow mode

This mode is a very important mode of Vieb, as it allows you to click on links or other elements using the keyboard. There are two versions with very different implementations, one for the current tab with f and one for new tab using Shift with f. These will be explained in separate paragraphs, as the underlying implementations are very different, even if the differences are not immediately visible to the user.

follow by click

After pressing f from normal mode, all clickable elements should be assigned a key next to them. After choosing the key, Vieb will emulate a click on the chosen element. For the regular follow mode, these come in four flavours:
  • Regular anchor tags (a) outlined in blue - These will be clickable in regardless of the href attribute's value, even for href values such as "#". The result will be no different from clicking a link using a regular browser.
  • Clickable input elements (checkboxes, radio and other buttons) outlined in red - These will be clicked on with no real differences compared to a regular browser.
  • Input fields (textarea and inputs) outlined in green - These will be focused by clicking on them. Unlike other follow mode types, Vieb will not go back to normal mode but enter Insert mode directly. This allows you to start typing in the input field after choosing it with follow mode. When in normal mode, you can also jump to the first text input using g i.
  • Onclick handlers outlined in orange - Elements with JavaScript event handlers will be made clickable. Both the onclick attribute and elements with event listeners are supported.
Clicking links with the mouse when using insert mode will mostly do the same thing.

follow in new tab

The Shift with f key allows you to open links in a new tab. Unlike the regular follow mode, only anchor tabs will be available to click on. Aside from that, all the href attributes are filtered. The filter will disable the keys for links with no valid url, as these couldn't be opened in a new tab. This means only anchor tags with valid urls will be made clickable. Finally these links will not be opened by emulating a mouse click, but by opening a new tab with the href attribute as the url.

Cursor mode

Cursor mode can be used to move "cursor" using the keyboard. It is required to enter cursor mode before going to visual mode. To enter cursor mode, press c, s or v when in normal mode. When in cursor mode, hover events will be triggered when moving the cursor. If the cursor is pointing at a url, the url will be shown at the bottom of the screen. Aside from that, images can be downloaded and urls can be copied. Lastly, cursor mode can be used to scroll inside scrollable elements smaller than the main window. The following actions are available by default in cursor mode:
  • Use h j k l to move the cursor around.
  • Press y to copy the current url under the cursor to the clipboard.
  • You can download the image or svg at the current position using d.
  • Execute a regular mouse click by pressing f.
  • Start inspecting at the current cursor location with e.
  • Click and enter insert mode at the cursor location using i.
  • For middle and right-click, use m and r respectively.
  • To jump a larger distance horizontally, use w and b.
  • Press g g or Shift with g to scroll to the top or bottom of the page.
  • Entering Shift with h or Shift with l will move the cursor to the top or bottom of the view.
  • To execute a scrolling action at the current position, use Shift with j or Shift with k.
  • The keys ^ and $ will move the cursor to the end or the beginning of the view.
  • Combining Ctrl with h j k l will move the cursor by one pixel in each direction.
  • To move fast up and down, use Ctrl with u and Ctrl with d.
  • Finally, pressing v when in cursor mode, will enter visual mode.
By default, F1 is mapped to this help page for this and all other modes, similar to the default binding to go back to normal mode (Escape or Ctrl with [). Some movemement actions will also scroll the window when reaching the bottom or top of the current view.

Visual mode

This is the only mode than can not be directly entered from normal mode, but is instead entered after pressing v from cursor mode. The main purpose of this mode is to select and copy text. The full list of options looks like this:
  • Use h j k l to move the cursor around.
  • To jump a larger distance horizontally, use w and b.
  • Press g g or Shift with g to scroll to the top or bottom of the page.
  • Entering Shift with h or Shift with l will move the cursor to the top or bottom of the view.
  • To execute a scrolling action at the current position, use Shift with j or Shift with k.
  • The keys ^ and $ will move the cursor to the end or the beginning of the view.
  • Combining Ctrl with h j k l will move the cursor by one pixel in each direction.
  • To move fast up and down, use Ctrl with u and Ctrl with d.
  • Finally, pressing y or c will copy the selection to the system clipboard.

Commands

In this chapter all the available commands will be listed and explained. By default, all supported commands will be autocompleted using a suggestions dropdown menu, similar to the nav mode suggestions. These suggestions can be disabled for commands by setting "suggestCommands" to false. The full list of commands is as follows:
  • help (or h)
  • version (or v)
  • downloads (or d)
  • history
  • accept (or confirm)
  • reject (or deny)
  • devtools (or dev)
  • set
  • reload (or r)
  • quit (or q)
Each paragraph in this chapter will explain a single command from the list above.

help

The help command can be used as an alternative for the F1 key. It will open the help in the current tab, unless you are already on a different website. Optionally, the help command takes a single argument, which can be used to refer to a section of this help page. For example, to navigate to the basics chapter: 
help basics
This command can also be used when this help page is already open, as it will simply move the page to the right section. The help command can be shortened by only using "h". All paragraphs are prefixed with the chapter they are in, and as a result the current paragraph can be opened directly with: 
h commands-help

version

The version command takes no arguments. It is used to diplay version information about Vieb. There are versions listed for Vieb and Chromium, and you can also navigate to the github repository from the version page. The version command is also available as "v".

downloads

The downloads command takes no arguments. The short version of this command is "d". This page can be used to view all the downloaded files. To learn more about the download page, different download methods and other special pages, check out the Special pages.

history

The history command takes no arguments. This command will open a list of all the website visisted. To learn more about Vieb's history settings and configurations, see history page info or the history chapter.

accept

This command can be used to confirm a download request. It can only be used if the "confirm" download method is used. When this mode is active, no files will automatically download, but you will be asked to accept or reject the file. This command can also be called as: 
confirm
On the downloads page you can view a list of downloaded files, and also view the current unconfirmed download.

reject

This command can be used to reject the current unconfirmed download. It's usually not required to call this command, because new download requests will replace the old one. This command can also be called as: 
deny

devtools

The devtools command takes no arguments. This command can be used to open to open the developer tools for the current website. The short version of this command is "dev". To open the Vieb developer/debug console, see the startup options.

set

This command has already been mostly explained in this paragraph. To summarize, this command can be used to set any setting during runtime, without saving it to the viebrc.json settings/config file. It is always in the form of: 
set setting value
For example, to increase the notification duration for the current session, use the following command (from 6 seconds to 10 seconds): 
set notification.duration 10000
If no notification is shown, the setting was successfully applied. Note that the duration and position of the notifications can only be changed if the Vieb notification is used instead of system notifications. Lastly, Vieb will automatically block websites with an invalid or missing certificate. To bypass this, either enter the url with the full protocol (http://), or change the following setting: 
set redirectToHttp true

reload

The reload command can be used to reload the settings from the settings file. It takes no arguments, required or optional. All previous "set" command changes are reset by using reload, so it can be used to undo any unwanted changes made using the set command. It can also be used to change the keybindings. or to change the settings permanently (instead of temporary using the "set" command). A short version of the reload command is just using "r".

quit

Last but not least is the quit command. This simple command takes no arguments, and is the proper way to quit Vieb. For beginners who haven't changed the related keybindings, these are the steps to quit Vieb regardless of mode:
  • Press Escape to go back to normal mode (Escape does nothing when already in normal mode).
  • Use Shift with ; or : to enter command mode.
  • Type "quit" into the navbar and press Enter.
The short version of the quit command is "q".

History

Your browsing, download and tab history are automatically stored when using Vieb with the default settings. Each of these parts have separate settings to configure how and if they should store your history.

browsing history

All websites you visit are stored in a "hist" file inside Vieb's config folder. Everytime a page is loaded, the hist file is appended with a single line. By default, suggestions will be displayed when in nav mode. These suggestions are entirely based on websites you have previously visited. The navigation bar will turn green when a suggestion is selected, cyan for direct navigations and orange for search. Webpages you visit often or those that match more closely with your search will appear at the top of the list. These suggestions can be disabled by setting "history.suggest" to false. To change or disable the browsing history, there are two other settings:
  • history.clearOnQuit - when this setting is set to true, the browsing history is cleared when quiting Vieb. This will not clear the currently opened tabs or the download history, but only influences if the browsing history should be cleared on exit.
  • history.storeNewVisits - when this setting is true, your browsing history is extended for every page you visit. This means that once a page is loaded, the url, the title and the date of the visit will be stored. When this setting is set to false, no new entries will be added for new site visits. This setting does not influence the previously stored history, or the download and tab history.

other history stored

The download settings are explained in the special page section. To learn more about the tab history settings, see the startup chapter. Because these settings are split between downloads, tabs and browsing history, you can finetune what you want to store and when to clear it automatically.

private mode

Having separate history settings means that there is no traditional private window available in Vieb, but by configuring the right settings, something equivalent can be achieved. An example for this based on the default settings can be something like this:
  • Set "downloads.removeCompleted" to true
  • Set "history.storeNewVisits" to false
  • Set "tabs.keepRecentlyClosed" to false
The combination of these settings means that once a tab is closed, or a download is completed, Vieb has no history stored in any of the three categories. You can of course expand these three settings with more strict settings, such as the option to clear any download regardless of progress when quiting, or the setting to never store the tabs at all by disabling the "tabs.restore" setting. By default, command mode will suggest and autocomplete settings that can be configured, which aims to make it easier to set these settings at runtime. Of course it's also possible to store these settings permanently in the viebrc config file. Both of these ways to change the settings are explained settings chapter.

Adblocker

Vieb includes an adblocker made by cliqz (see mentions for details). By default, the easylist and easyprivacy filter lists are included and enabled with Vieb. The lists will only be updated when a new Vieb released, unless you change the adblocker setting. The adblocker setting can only be changed using the viebrc config file. There are four different options for the adblocker setting:
  • off - The adblocker won't be loaded, and no ads will be blocked
  • static (default) - The adblocker will be loaded and block ads using built-in lists, but won't automatically update the easylist and easyprivacy filters. The lists will only be updated when downloading a new version of Vieb from the releases page. Additional lists can still be added as explained below.
  • update - The adblocker will be loaded and block ads using built-in lists, but also automatically download the latest easylist and easyprivacy filter lists on startup. This is not the default, because connections to a third-party site should be opt-in. Additional lists can still be added as explained below.
  • custom - The adblocker will only load when lists are added manually. This option should be used in case you don't want to use the easylist and easyprivacy lists, but still want Vieb to block ads and/or trackers based on other filter lists. If this option is chosen after Vieb was started with this setting on "static" or "update", you should check the "blocklists" folder for lists you might not want to use.
Regardless of the chosen option, you can add new lists to the "blocklists" subfolder inside the Vieb config folder, but only "custom" allows you to remove the easylist and easyprivacy filter lists. In case you want to add a different filter list, make sure it has the right extension and syntax, or go to easylist.to for more information about these lists.

Special pages

Vieb has a couple of special pages that can be accessed like so "vieb://help". Most of these pages also have a dedicated command to navigate to them, but using nav mode to go there is possible as well. Each special page will be explained below.

help page

This is the current page and also a complete reference of Vieb's functionality. The location of this special page is "vieb://help". You can link to specific section of this page using "vieb://help#basics", or by calling the help command with extra arguments.

downloads page

This page displays a list of all the downloaded files using Vieb. The list is stored between sessions by default, but there are two related settings to change this behaviour:
  • downloads.removeCompleted - This setting can be set to true in case you want to remove all completed downloads from the list. You will still get a notification when a download is done, but it won't stay in the download history. Progressing or cancelled downloads will still be listed, even between sessions.
  • downloads.clearOnQuit - This setting can be set to true if you want to clear the list on quit. When set to true, no download history will be saved to disk. The tab and browsing history are configured separately.
There are three different download methods in Vieb (setting: "downloads.method"):
  • automatic (default) - which will download all files automatically similar to chromium, the configured "downloads.path" will be used to store all downloads.
  • ask - when this mode is active, the download path will not be used, but instead a file explorer will be displayed to ask for the download location.
  • confirm - in this mode the :accept and :reject commands are used to confirm a download. On the downloads page the current unconfirmed download is shown. As with the automatic download setting, the "downloads.path" location is used to store the download.
All downloads regardless of mode will be displayed on the downloads page, which can also be accsed as "vieb://downloads" or with the downloads command.

version page

This page contains the current Vieb version and a button to check for updates. It can be accessed with the :version command (:v), or as "vieb://version".

history page

On this page you can find all your browsing history. The page is sorted by date and contains breakpoints such as "Today" or "This hour", which you can jump to directly using the links at the top. There is also the option to delete specific pages from your history, or you can delete an entire day, week or everything. The history page can be opened using the history command, or as "vieb://history". To change the history settings, see the history chapter.
